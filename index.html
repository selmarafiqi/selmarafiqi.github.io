RSP Campaign Visualizer

Website
Check Out The Repository on GitHub

Overview

NASA GISS has given me the opportunity to work with the RSP team to build and design a website that is primarily used to plot data on the map. For this project, we used Visual Studio Code, HTML, CSS JavaScript, JQuery, mustache.js, and JSON.

A major assist in figuring out how to design this website was that NASA provided us a web design system that included UI components and even page templates. This helped me choose the right features for our team's specific website.


Project Details

  • Role: Web Designer/Developer Intern
  • Duration: Jun 2019-August 2019
  • Context: Web Application
  • Client: NASA GISS

The Problem

The web frame my team and I had come up with showed that the "Campaigns" dropdown would already be expanded. Knowing that there was a lot of data and flights associated with each campaign, this dropdown would not look pleasing to a user.

  1. There are too many data for the the dropdown to be kept expanded.
  2. How can we organize the dropdown to look pleasing to the user while creating easy user experience?

The Solution

I came up with the idea of placing the "Campaign" tab inside the dropdown box which acts as an expand and collapse functionality to show a clean and simple webpage.

  1. Talked this idea over with our team's Software Engineer
  2. Used JavaScript to allow the to content to expand/collapse when a checkbox is clicked.
  3. Shortened the dropdown box and added a scroll functionality.

Feedback to self

Given the two months time constraint, moving the clear button out of the dropdown box would have been my next step. Whereas when a user clicks a Campaign, the clear button gets pushed to the bottom of the list, forcing the user to scroll down.

Overview

When I was first approached to design a website for this fast-food restaurant, I started to feel overwhelmed because I had not done such a website before.

With the help of Luminal Site Builder, I was able to come up with a simple, fun, and engaging website. Although I did have issues whereas I did not know what to do with the menu. When I completed my first design, I knew it was not good. The menu did not look easy on the eye as the client had many items to list. So I decided to place them into categories and add a corresponding photo resulting in a great look.

As I completed the website design, my client loved it! But soon I realized this design may not be the best for the future. Currently, the website is built on just one HTML page. If my client decides to add more information, placing more on the page can cause much more loading time. I talked to my client about this issue and they did not want to change it as they loved it as is.
